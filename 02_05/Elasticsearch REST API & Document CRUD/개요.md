# Elasticsearch의 데이터 교환
- 방식
    - 분산 검색 및 분석 엔진으로, 대량의 데이터를 빠르게 검색 및 분석 사용
    - RESTful API를 통해 클라이언트와 통신
- REST API 특징
    - HTTP 기반 동작
    - 자원을 URL로 표현
    - HTTP 메서드를 활용하여 CRUD 작업 수행
    - JSON, XML 등 다양한 데이터 형식을 사용

# Elasticsearch 문서 업데이트
- 문서는 불변하므로 직접 수정되지 않음, 업데이트 시 새로운 문서로 저장 등으로 사용 가능
- 업데이트 과정
    1. 기존 문서 조회
    2. 변경 사항 적용
    3. 새 문서 색인
    4. 이전 문서 삭제 처리
    5. 세그먼트 병합

# Elasticsearch에서 세그먼트와 Flush의 관계
1. upsert & update는 기존 문서를 수정하는 것이 아니라 새로운 세그먼트를 생성하는 방식으로 동작
2. Flush는 새로운 세그먼트를 디스크에 기록하는 과정
3. 세그먼트가 증가하면 자동으로 병합 수행
