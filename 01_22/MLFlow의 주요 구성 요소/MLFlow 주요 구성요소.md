# **MLFlow 주요 구성 요소 및 특징 정리**

MLFlow는 머신러닝 라이프사이클 전반을 관리하기 위한 오픈소스 플랫폼입니다. 각 핵심 구성 요소와 특징을 정리했습니다.

# **1\. MLFlow Tracking**

머신러닝 실험을 체계적으로 관리하기 위한 API 및 UI를 제공하는 컴포넌트입니다.

## **주요 기능**

* **Parameter (파라미터)**: 모델 학습에 사용된 입력 데이터들을 키-값(Key-Value) 쌍으로 저장 및 추적합니다.  
* **Metric (메트릭)**: 모델의 성능 지표(정확도, 손실 함수 등)를 숫자 형태 및 그래프로 시각화합니다.  
* **Artifact (산출물)**: 모델 파일, 이미지, 데이터 파일 등 실험 결과물을 파일 형태로 저장합니다.  
* **Code Version**: Git Commit 해시 등을 기록하여 실행된 코드의 버전을 추적합니다.  
* **Tags**: 실험을 설명하는 커스텀 라벨을 추가할 수 있습니다.

## **환경 구성 (3가지)**

1. **Localhost (Default)**: 로컬 호스트에 모든 기록을 저장합니다.  
2. **Localhost with Various Data Stores**: 로컬 디렉토리나 데이터베이스(SQLite 등)에 기록을 저장합니다.  
3. **Remote Tracking Server**: 별도의 서버를 구축하여 여러 사람이 공유하는 기록 저장소로 활용합니다. (S3, GCS 등 스토리지 연동 가능)

# **2\. MLFlow Projects**

머신러닝 코드를 재사용 가능하고 재현 가능한 형태로 패키징하기 위한 표준 형식입니다.

## **특징**

* 각 프로젝트는 코드, 데이터, 환경 설정 등을 포함하는 디렉토리 또는 Git 저장소로 구성됩니다.  
* MLproject 파일 내에서 실행 스크립트와 파라미터 정의가 가능합니다.  
* 다양한 실행 환경(Local, Docker, Conda 등)을 지원합니다.

# **3\. MLFlow Models**

다양한 머신러닝 모델을 일관된 방식으로 패키징하고 배포하기 위한 형식입니다.

## **특징**

* **Flavor**: 모델이 어떤 형식(sklearn, pytorch, keras 등)으로 저장되었는지 나타내는 속성입니다.  
* **저장 구조**: MLmodel 파일(메타데이터)과 실제 모델 가중치 파일(.pkl, .pth 등)을 포함하는 디렉토리 형태입니다.  
* 저장된 모델은 하나의 명령어로 즉시 Serving(REST API 등)이 가능합니다.

# **4\. MLFlow Model Registry**

머신러닝 모델의 전체 생애주기를 체계적으로 관리하기 위한 중앙 집중형 저장소입니다.

## **주요 기능**

* **Model Lineage**: 모델이 어떤 실험과 실행에서 생성되었는지 추적합니다.  
* **버전 관리**: 특정 모델의 여러 버전을 체계적으로 관리하고 비교합니다.  
* **단계 전환**: 모델의 상태를 Staging → Production → Archived 단계로 관리합니다.  
* **주석 및 태그**: 모델에 대한 설명이나 특정 정보를 태그로 관리합니다.

# **5\. MLFlow 아키텍처 및 워크플로우**

1. **실행(Run)**: 실험 진행 시 Tracking Server에 파라미터, 메트릭, 산출물을 자동 저장합니다.  
2. **등록(Register)**: 테스트가 완료된 모델을 Model Registry에 등록합니다.  
3. **관리(Manage)**: 리뷰 및 승인 절차를 거쳐 배포 단계를 설정합니다.  
4. **배포(Deploy)**: REST API 서버 혹은 CI/CD 툴을 통해 실무 환경에 배포합니다.

# **6\. MLOps 도구 비교 (MLFlow vs 기타)**

| 항목 | MLFlow | Kubeflow | Metaflow | W\&B |
| :---- | :---- | :---- | :---- | :---- |
| **출시 주체** | Databricks | Google | Netflix | Weights & Biases |
| **주요 기능** | 실험 추적, 모델 관리, 배포 | 파이프라인, 서빙, 워크플로우 | 파이프라인, 스토리지 관리 | 실험 추적, 시각화, 협업 |
| **설치 난이도** | 쉬움 (로컬 가능) | 복잡 (Kubernetes 필요) | 중간 (로컬 & 클라우드) | 쉬움 (클라우드 기반) |
| **모델 배포** | 기본 제공 (REST API) | 강력 (KFServing 등) | 지원 (외부 연동 필요) | 지원하지 않음 |
| **프레임워크** | 매우 범용적 | TensorFlow 중심 | Python 중심 | 매우 범용적 |
| **장점** | 가볍고 직관적, 파이썬 기반 최적화 | 강력한 컨테이너 기반 자동화 | 데이터 사이언티스트 친화적 | 깔끔한 UI와 강력한 시각화 |
